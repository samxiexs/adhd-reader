# Focus Reader · ADHD 便利阅读器

一个无账户、默认不保存内容的网页阅读工具。它保持原文措辞与语言不变，只通过更清楚的段落节奏、层级与有限重点标注，让长文本更容易开始阅读。

## 当前功能

- 粘贴纯文本或保留基础富文本（粗体、斜体、链接、列表）。
- 本地规则模式无需模型密钥；AI 模式使用服务端的 OpenAI 兼容接口。
- 结果可直接编辑，并复制为富文本或下载 Markdown。
- 支持 Markdown 和含可选文字层的 PDF；默认原文/便利阅读版分屏，也可单栏阅读。
- 文本和文件仅在当前浏览器会话与请求处理期间存在，不写入数据库或文件存储。

## 本地运行

```bash
npm install
npm run dev
```

打开终端显示的本地地址即可使用。未设置 AI 环境变量时，本地规则模式仍可完整使用。

## AI 服务商配置

复制 [`.env.example`](.env.example) 为 `.env.local`，填入一个 OpenAI 兼容服务商的环境变量。前端仅会收到服务商显示名称、ID 与模型名；密钥始终只在服务器运行环境中读取。

AI 接口只接收段落 ID 与原文文本，并只允许返回重点的字符范围。服务端会验证每一个 ID、范围和段落集合；任何缺段、重复段或无法验证的结果都会被拒绝，继而可回退到本地模式。

## Docker 部署

```bash
docker build -t focus-reader .
docker run --rm -p 3000:3000 --env-file .env.local focus-reader
```

容器不包含数据库或持久卷。请在部署平台的运行环境中管理模型密钥，不要把 `.env.local` 提交到代码库。

## 限制

- 当前 PDF 仅支持含可选择文字层的文档；扫描件与图片型 PDF 会提示暂不支持 OCR。
- AI 服务商首发范围为 OpenAI 兼容 API；非兼容服务需要新增适配器。
- 该工具改善阅读呈现，不用于 ADHD 的诊断或医疗建议。
